Micro-casing is a mobile app that replaces $200/hour coaching with 400-second gamified case studies, making consulting interview prep fast, affordable, and sustainable for anyone competing for a spot at McKinsey, BCG, or Bain.

Traditional prep is slow, expensive, and hard to stick to. Micro-casing flips the model: bite-sized cases on your commute, built on the same decision-making framework top coaches use.

The result: 2.84% download-to-paid conversion by day 35, 8 active subscription plans, and +700% subscriber growth since launch, driven entirely by freemium onboarding.

The Challenge

Defining the
Product System
and Brand Guide

Micro-casing had been built from AI-generated wireframes with no design system behind it. The UX audit mapped four specific failures, each one blocking a different part of the product.

Fragmented visual identity. No shared tokens, no hierarchy logic, no component consistency. Every screen made the user rebuild context from scratch, creating cognitive load before a single case was attempted.

Broken navigation and user control. Users were locked into a single forced flow. No bottom nav, no way to backtrack, no access to profile or filtering. The product grew its case library weekly but gave users no way to orient inside it.

Misaligned copy and tone of voice. Gaming language and consulting authority were competing in the same interface. The result was a product that felt neither credible enough for serious prep nor approachable enough to open daily.

Cognitive overload at critical touchpoints. The case results screen, the moment where users decide to continue or quit, was cluttered with redundant information. Drop-off was happening exactly where engagement needed to peak.

01

Mini UI Kit cut implementation ambiguity. A component library with colours, typography, and iconography gave engineers a single source of truth. No guessing specs, no revision loops. The team shipped faster because every pattern was already defined and reusable.

02

High-fidelity prototype aligned the team before a line of code was written. Eight complete flows with interaction logic let developers see exact behaviour, transitions, and edge cases upfront. Decisions that would have surfaced mid-sprint were resolved in Figma.

03

Scalable content architecture removed design as a bottleneck. A filtering system built around difficulty, category, and new releases means new cases plug directly into the existing structure. The content library can grow every week without requiring a design intervention.

Complexity

Where It Got Hard
and How We Moved

01
Building a system inside a sprint.
Two weeks is not enough time to build a proper design system — but handing engineers a Figma file with no components would have created permanent debt. The answer was to define only the component set that appeared in production screens: a minimal but real kit, not a comprehensive library. It gave the team a foundation without killing the sprint.
02
Gamification vs. credibility in a high-stakes context.
Consulting interviews are serious. Gamification signals casual. Design too playful and a competitive McKinsey candidate won't trust it; design too corporate and they won't open it daily. The resolution was to separate tone from visual weight: the interface stayed clean and structured while the copy carried the friendly register, and the Casey mascot was introduced as a directional hypothesis to lower the perceived barrier to entry.
03
Rebuilding on top of an AI-generated file.
The inherited Figma file had been generated by AI and built out by engineers — no auto-layout, no components, no naming logic. Starting from scratch would have been faster, but would have forced the team to re-implement screens already in production. Instead, the existing architecture was used as a map, not a template: each screen was rebuilt with proper component logic while preserving the navigation structure the team already understood and shipped.

Lessons Learned

What This Project
Taught Me

01
UX Leads, UI Follows
Flows can always evolve and improve. Visual identity, once defined, becomes the stable foundation everything else builds on.
02
Human Direction Makes AI Meaningful
AI accelerated exploration and iteration, but every strategic and creative decision required human judgment to be meaningful.
03
Communication Is a Design Deliverable
Structured documentation and clear client communication were as critical as the design itself. Without them, even great work loses direction.
04
Prototype Early, Prototype Always
Prototyping was the single most effective way to validate decisions, communicate intent, and reduce misalignment with the development team.
05
Workflow Is Efficiency
Defining a clear personal workflow before starting saved time, reduced errors, and kept the project moving within a strict two-week sprint.
06
AI Is Not a Trend, It Is Infrastructure
AI tools are now standard infrastructure. Using them effectively is a baseline professional expectation, not a differentiator.
